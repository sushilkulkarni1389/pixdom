# temp-file-security — requirements

### Requirement: Restrictive permissions on temp directory
The animated renderer SHALL set permissions `0o700` on the temp directory immediately after creating it (`fs.mkdir`). This prevents other local users from reading intermediate frame files.

#### Scenario: Temp directory created with 0o700 permissions
- **WHEN** `renderAnimated()` is called
- **THEN** the per-call temp directory has permissions `0o700` (owner read/write/execute only)

### Requirement: Signal-handler cleanup for temp files
The animated renderer SHALL register `process.on('SIGTERM', cleanup)` and `process.on('SIGINT', cleanup)` handlers before beginning frame capture. The cleanup handler SHALL synchronously delete the temp directory via `fs.rmSync(tmpDir, { recursive: true, force: true })` and re-raise the signal so the process exits normally. Handlers SHALL be removed after `renderAnimated()` completes to avoid accumulating listeners across multiple calls.

#### Scenario: SIGTERM during frame capture removes temp dir
- **WHEN** SIGTERM is sent to the process while `captureFrames` is in progress
- **THEN** the temp directory and all captured frame files are deleted before the process exits

#### Scenario: Signal handler removed after render completes
- **WHEN** `renderAnimated()` returns (success or failure)
- **THEN** the SIGTERM and SIGINT listeners added during that render are removed

### Requirement: Temp file names are unpredictable
All temp files (frame files, palette file, intermediate encode output) SHALL use the random directory name (`pixdom-<uuid>`) as their containing directory. No temp file SHALL use a predictable name outside of the random directory.

#### Scenario: Temp directory uses UUID-based name
- **WHEN** `renderAnimated()` is called
- **THEN** the temp directory path is `<os.tmpdir()>/pixdom-<uuid>` where `<uuid>` is generated by `randomUUID()`

#### Scenario: Intermediate files inside random directory only
- **WHEN** animated rendering completes
- **THEN** no files named `pixdom-*.png`, `palette.png`, or `*.gif`/`*.mp4`/`*.webm` exist in `os.tmpdir()` directly (only inside the UUID subdirectory)
